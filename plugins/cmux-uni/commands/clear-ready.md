# /clear-ready — 클리어 준비

현재 세션 컨텍스트를 저장하고, `/clear` 후 즉시 재개할 수 있는 프롬프트를 생성합니다.

트리거 키워드: "클리어 준비", "clear 준비", "clear ready", "/clear-ready"

---

## 실행 절차

### 1단계: 상태 수집 (병렬 읽기)

다음 항목을 읽어 현재 상태 파악:

1. **PDCA 상태**: 현재 프로젝트의 `.bkit/state/pdca-status.json` (있으면)
2. **현재 메모리**: 프로젝트 memory 디렉토리의 `MEMORY.md`
3. **활성 Plan 문서**: `docs/01-plan/features/` 디렉토리의 최신 파일 (있으면)
4. **활성 Design 문서**: `docs/02-design/features/` 디렉토리의 최신 파일 (있으면)

### 2단계: 메모리 업데이트

이번 세션에서 새로 알게 된 중요 정보를 메모리에 저장:

- **project 타입** 저장 대상: 현재 진행 중인 Phase, 다음 작업 명령어, 핵심 기술 결정사항
- **feedback 타입** 저장 대상: 이번 세션에서 발견한 패턴, 주의사항, 잘 작동한 접근법

### 3단계: 재개 프롬프트 생성

아래 형식으로 재개 프롬프트를 생성하여 **코드 블록으로 출력** (복사하기 쉽게):

```
---
## 세션 재개 컨텍스트 — {오늘날짜} {현재시간}

**프로젝트**: {프로젝트명}
**경로**: {작업 디렉토리 전체 경로}
**마지막 완료 작업**: {직전에 완료한 작업 1줄 요약}

### 즉시 실행할 다음 작업
{구체적인 다음 명령어 또는 작업 설명}

### 핵심 컨텍스트
{이번 세션에서 중요한 기술 결정, 제약사항, 주의사항을 불릿으로}
- ...

### 미완료 / 정리 필요 항목
{아직 완료되지 않은 작업 또는 stale 상태 항목}
- ...
---
```

### 4단계: 안내 출력

프롬프트 출력 후 다음 안내 표시:

```
✅ 메모리 저장 완료
📋 위 프롬프트를 클립보드에 복사하세요
🔄 /clear 실행 후 복사한 프롬프트를 붙여넣으면 즉시 재개됩니다
```
